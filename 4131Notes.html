A closure allows a function to be able to hold onto certain values ,such as free variables, which are variables that are defined locally, but exist within an enclosed scope. Without a closure, these variables are set to references, so when it is used later on, the value of the variable changes to what it was last modified to. Closure will keep a record of all the current scope it is wrapped in, so in a for loop without another closure to limit the scope, the changes made to any variable in the scope will permeate to other closures made in the scope. We want to create a scope where there is no change the variable, so putting a new scope inside the for loop will save the values of the current loop, since the values are encasulated in a scope If you define a closure within a scope, and the value of the variables that the closure saves does not change within the scope the closure is in, then the value of the variables will not change either outside scope of the closure should be static, no chance to change
Event Listeners are added via javascript. Will add the event handler to the button by getting the id of the button. Based on the event the event listener is listening for, the event handler will execute a function. The event handler is the function that is called after the event has been detected. The event can be captured either in the bubbling phase or the capture phase. Set useCapture to true if you do not want bubbling, like an event especially for a child element Bubbling makes sure the child element's event is triggered first rather than capturing, where the parent's event is triggered first
Examples of Events and Event handlers are click onClick change onChange focus onFocus load onLoad mouseover onMouseOver select onSelect
Event travels from the DOM to the target element, triggers on the target element, and sends the response back up to the DOM, when an even is fired onto an element, it is delivered onto the element's event handler, then the element's parent event handler, etc. To make sure it only happens on the element you want, Remove an event listener by calling removeEventListener with the same arguments that you passed to addEventListener to register the event handler

- TCP/ip is the network that http sends request and reponses through. - IP obtains the address and TCP guarantees delivery of data to that address. - TCP is the transport protocol, IP is the network PROTOCOL, dealing with addressing and routing - DNS converts IP to URL Consists of 2 protocols, TCP and UDP - TCP is responsible for establishing connection between 2 machines, then transporting all the information from one device to another - TCP is reliable, acknowledgement is expected, will resend if not recieved by receiver - UPD is not reliable, but is used when reliability is not critical - IP is a network layer protocol, each machine in an IP network has an ip address, and the IP software is responsible for routing a message from the source IP to the destination IP - In an ip machine, TCP supports many sockets and ports. - In an IP network, each machine has an IP address
STATUS CODES: 404 --> NOT FOUND 200 --> OK 405 --> NOT ALLOWED 403 --> FORBIDDEN 1xx (Informational): Request received, server is continuing the process. 2xx (Success): The request was successfully received, understood, accepted and serviced. 3xx (Redirection): Further action must be taken in order to complete the request. 4xx (Client Error): The request contains bad syntax or cannot be understood. 5xx (Server Error): The server failed to fulfill an apparently valid request.
